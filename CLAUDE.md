## 시작 시 필독
- 대화 시작 시 `tickets/index.md`를 읽고 프로젝트 현황을 파악한다

## 소통 언어
- **모든 대화는 한국어로** 진행한다. 코드 외 모든 텍스트는 한국어로 작성한다.

---

## 티켓 워크플로우

티켓은 `tickets/` 폴더의 `.md` 파일로 관리한다.

### 티켓 단위 규칙
- CRUD는 기능 단위로 **하나의 티켓**으로 묶는다.
- 티켓을 지나치게 잘게 쪼개지 않는다.

### 파일명 규칙
- 티켓 `.md` 파일명은 주요 기능을 요약한 짧은 영문 타이틀로 작성한다.
- 예: `todo-add.md`, `login.md`, `edit-modal.md`

### 날짜 규칙
- 형식: `YYYY/MM/DD HH:mm`
- 상태 변경 시 시작/완료 날짜 2개 기록

### 완료 처리 규칙
1. 해당 `.md` 파일의 상태값을 `완료`로 업데이트
2. 완료 날짜 기록
3. 파일을 `tickets/done/`으로 이동

### 폴더 구조
```
tickets/
├── [진행 중 또는 대기 티켓].md
├── index.md
└── done/
    └── [완료된 티켓].md
```

### 상태 정의
| 상태 | 설명 |
|------|------|
| 대기 | 아직 작업 시작 전 |
| 진행 중 | 현재 작업 중 |
| 완료 | 작업 완료 → `tickets/done/`으로 이동 |
| 보류 | 외부 사유로 일시 중단 |

## Git 규칙
- 커밋 메시지는 Claude가 작성하고, **실제 커밋은 사용자가 직접** 한다
- 커밋 메시지는 `tickets/git_commits/YYYY-MM-DD.md`에 누적 저장한다
- 형식: `## #순번 — YYYY/MM/DD | 제목`
- 파일 상단에 `# 커밋 내역` 헤더 포함
- 각 커밋은 제목 아래 `-`로 변경 내용 나열, 중간 타이틀 없이

## 코드 스타일
- **들여쓰기**: 스페이스 대신 **탭(tab)** 사용
- **탭 너비**: 2칸

## 클래스 네이밍 규칙 (CSS/SCSS)
- **스네이크 케이스** 사용: 단어 사이는 언더바(`_`)로 연결
- 부모 이름을 무조건 다 붙이지 않고, 맥락상 파악 가능하면 핵심 단어로 축약
- `__`는 구조적 구분이 꼭 필요한 복합 요소에만 사용

## 주석 규칙
### SCSS
- 큰 섹션은 구분선 타이틀, 중간 단위는 한 줄 주석, 불필요한 주석 금지
### JSX
- 간단한 한 줄 한국어 주석만 허용, 흐름 핵심마다 한 줄씩
